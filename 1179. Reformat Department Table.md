```sql
SELECT
    id,
    -- January Revenue
    SUM(CASE WHEN month = 'Jan' THEN revenue ELSE NULL END) AS Jan_Revenue,
    -- February Revenue
    SUM(CASE WHEN month = 'Feb' THEN revenue ELSE NULL END) AS Feb_Revenue,
    -- March Revenue
    SUM(CASE WHEN month = 'Mar' THEN revenue ELSE NULL END) AS Mar_Revenue,
    -- April Revenue
    SUM(CASE WHEN month = 'Apr' THEN revenue ELSE NULL END) AS Apr_Revenue,
    -- May Revenue
    SUM(CASE WHEN month = 'May' THEN revenue ELSE NULL END) AS May_Revenue,
    -- June Revenue
    SUM(CASE WHEN month = 'Jun' THEN revenue ELSE NULL END) AS Jun_Revenue,
    -- July Revenue
    SUM(CASE WHEN month = 'Jul' THEN revenue ELSE NULL END) AS Jul_Revenue,
    -- August Revenue
    SUM(CASE WHEN month = 'Aug' THEN revenue ELSE NULL END) AS Aug_Revenue,
    -- September Revenue
    SUM(CASE WHEN month = 'Sep' THEN revenue ELSE NULL END) AS Sep_Revenue,
    -- October Revenue
    SUM(CASE WHEN month = 'Oct' THEN revenue ELSE NULL END) AS Oct_Revenue,
    -- November Revenue
    SUM(CASE WHEN month = 'Nov' THEN revenue ELSE NULL END) AS Nov_Revenue,
    -- December Revenue
    SUM(CASE WHEN month = 'Dec' THEN revenue ELSE NULL END) AS Dec_Revenue
FROM
    Department
GROUP BY
    id;
```

## ðŸ’¡ Query Explanation (Pivoting)

1.  **`GROUP BY id`**: This groups all records belonging to the same department, ensuring we get a single output row for each unique department ID.
2.  **`SUM(CASE WHEN month = 'MonthName' THEN revenue ELSE NULL END)`**: This is the core of the pivot operation:
      * For each department group, the `CASE` statement checks the value of the `month` column.
      * If the month matches (e.g., `'Jan'`), it returns the `revenue` value for that row.
      * If the month does **not** match, it returns `NULL`.
      * The **`SUM()`** aggregate function then adds up all the returned values. Since there is a primary key constraint on `(id, month)`, there will only be one non-null `revenue` value per month within each department group, so `SUM(revenue)` simply returns that single revenue amount. If all values are `NULL` (meaning no revenue was recorded for that month), the sum correctly results in `NULL`.

This effectively transforms the rows (months) into columns (`Jan_Revenue`, `Feb_Revenue`, etc.).
