# Problem 1084: Sales Analysis III
![Easy](https://img.shields.io/badge/Level-Easy-brightgreen?style=for-the-badge) [![Problem Link](https://img.shields.io/badge/LeetCode-Link-orange?style=for-the-badge&logo=leetcode)](https://leetcode.com/problems/sales-analysis-iii/description/)

```sql
SELECT
    p.product_id,
    p.product_name
FROM
    Product p
JOIN
    Sales s ON p.product_id = s.product_id
GROUP BY
    p.product_id, p.product_name
HAVING
    -- 1. Check that the MINIMUM sale date is within Q1 2019
    MIN(s.sale_date) >= '2019-01-01'
    -- 2. Check that the MAXIMUM sale date is within Q1 2019
    AND MAX(s.sale_date) <= '2019-03-31';
```

---

## ðŸ§ Query Explanation

### 1. `SELECT p.product_id, p.product_name`

Selects the **product ID** and **product name** for the final result.


### 2. `FROM Product p`

Uses the **Product** table as the base table with alias **`p`**.


### 3. `JOIN Sales s ON p.product_id = s.product_id`

Performs an **INNER JOIN** with the **Sales** table using `product_id` to connect each product with its sales records.


### 4. `GROUP BY p.product_id, p.product_name`

Groups all sales records **per product**, so aggregate functions can be applied for each product.


### 5. `MIN(s.sale_date)`

Finds the **earliest sale date** for each product.


### 6. `MAX(s.sale_date)`

Finds the **latest sale date** for each product.


### 7. `HAVING MIN(s.sale_date) >= '2019-01-01'`

Ensures the **first sale of the product** is **not before Q1 2019**.


### 8. `AND MAX(s.sale_date) <= '2019-03-31'`

Ensures the **last sale of the product** is **not after Q1 2019**.
